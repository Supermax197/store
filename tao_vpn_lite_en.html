Tao VPN Lite github:https://github.com/Supermax197/tao-vpn-lite

Tao VPN Lite can build private network on public network, make encrypted communication, and enable users to access remotely.

Tao VPN Lite is developed by Java, which is very, very simple, easy to use VPN software and across almost all platforms. No configuration, one key deployment.

Tao VPN Lite supports the HTTP and HTTPS protocols. That is to say, it support almost all websites, online games and applications.

Tao VPN Lite is developed by Java, which means that almost all platforms can run it, such as windows, Linux, Mac OS, Android, and so on. The client and server can run on different platforms.

Tao VPN Lite has its own unique communication encryption system, so it is incompatible with the traditional VPN software, which makes unique network security and penetration.

The use of Tao VPN Lite is very simple. Download the source code, compile it or download the jar package directly, and start the client's command.：
java -cp tao-vpn-lite.jar org.tao.vnp.lite.client.Client xxx.xxx.xxx.xxx 8082 8081
, XXX is the IP address of the VPN service side, the 8082 is the port number of the VPN service side port, and the 8081 is the port number of the VPN client side. Start the server-side command：
java -cp tao-vpn-lite.jar org.tao.vpn.lite.Server 8082
,the 8082 is the port number of the VPN service side port

On the Linux server, you can use similar deamon shell to start the server side.，
#! /bin/sh
while true ; do
NUM=`ps aux | grep -w tao-pry-1.0.jar | grep -v grep |wc -l`
if [ "${NUM}" -lt "1" ];then
echo "tao-pry-1.0.jar was killed"
nohup java -cp tao-vpn-lite.jar tao-vpn-lite.jar org.tao.vpn.lite.Server 8082 >/dev/null 2>%261 %26 fi
sleep 5s
done
exit 0

When both the client and the server started, user need to set up the local network agent to point to the IP address and port of the VPN client. Note that only the communication between the VPN client and the VPN server is encrypted, so the VPN client is best to run on your machine.
